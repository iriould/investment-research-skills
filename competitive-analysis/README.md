# Competitive Analysis / 竞对分析

一级市场 PE 投资竞对分析。搜索竞品信息，构建六维画像对比，评估竞争威胁，输出竞对分析报告。

## When to Use

用户说"竞对分析"、"竞品分析"、"竞争分析"、"competitive analysis"、"竞品对比"时使用。

## Quick Start

```
<目标公司名称> <竞对公司名称列表，逗号分隔>
```

示例：
```
智谱AI 月之暗面,百川智能,MiniMax
```

## Workflow

| Step | 内容 | 说明 |
|------|------|------|
| 0 | 读取配置与协作模式 | 搜索工具、协作模式（本地/飞书） |
| 1 | 确认分析范围 | 目标公司、竞对列表、分析深度 |
| 2 | 行业特征发现 | 识别关键成功因素（KSF）、维度权重分配 |
| 3 | 信息搜索 | 基础信息搜索 + 技术深度搜索 |
| 4 | 六维画像构建 | 产品/技术/商业模式/团队/融资/财务 |
| 5 | 对比分析与威胁评估 | 维度对比 + 技术深度对比 + 威胁评级 |
| 6 | 生成报告 | 结构化竞对分析报告 |

### Core Highlight: Six-Dimension Portrait Model

| 维度 | 默认权重 | 关键指标 |
|------|---------|---------|
| 产品 | 20% | 核心产品、产品形态、目标客户、功能覆盖 |
| 技术 | 15% | 技术栈、核心技术壁垒、开源项目、专利 |
| 商业模式 | 20% | 收费模式、定价、客户规模、渠道 |
| 团队 | 15% | 创始人背景、团队规模、核心人才 |
| 融资 | 15% | 最新轮次、估值、投资方、融资节奏 |
| 财务 | 15% | 营收规模、增长率、盈利状态、现金流 |

权重根据行业特征发现结果动态调整。

### Tech Sub-Dimensions (6 Sub-Dimensions, 5-Star Rating)

技术维度从简单的"技术栈+壁垒"扩展为 6 个深度子维度：

| 子维度 | 评估内容 |
|--------|---------|
| 技术架构 | 单体/微服务/事件驱动/混合，架构先进性和扩展性 |
| 技术栈 | 语言/框架/基础设施选择，技术栈成熟度和生态 |
| 开源 vs 专有 | 开源依赖比例、自研核心比例 |
| 研发投入 | 研发团队规模、研发支出占比 |
| 技术成熟度 | 从 POC 到大规模生产的成熟度 |
| 技术路线图 | 未来技术演进方向的清晰度和可行性 |

### Threat Assessment

| 威胁等级 | 条件 | 投资建议 |
|---------|------|---------|
| 🔴 高威胁 | 综合评分 ≥ 目标 且 产品高度重叠 | 需差异化突围，否则投资风险高 |
| 🟡 中威胁 | 部分维度领先 但 整体不及 | 关注动态，利用对方弱项 |
| 🟢 低威胁 | 整体弱于目标公司 | 保持优势，防止追赶 |

## Output

1. **竞争格局概览** — 行业竞争态势和关键发现
2. **六维画像对比表** — 所有公司的维度评分对比
3. **技术深度对比** — 技术 6 子维度的详细对比
4. **威胁评估矩阵** — 每个竞对的威胁等级
5. **投资建议** — 基于竞争分析的投资建议
