# 搜索源路由规则

本文件是竞对分析的搜索路由补充说明。具体工具选择必须优先遵循 `search-policy.md`，不要绑定某个固定搜索工具或 MCP server。

## 配置优先级

1. 读取 `config/investment-research-config.json`
2. 读取本 skill 运行期搜索配置
3. 如果配置为 `auto`，使用当前代理环境（Claude Code / Codex）中已配置、已授权、可调用的搜索工具

搜索工具按能力分组，而不是按工具名硬编码：

| 能力 | 适用场景 |
|------|----------|
| 通用网页搜索 | 公司概况、融资、团队、产品、客户、媒体报道、行业报告 |
| 创投/产业信息搜索 | 融资历史、估值、投资方、产业链、竞品排名 |
| 技术文档搜索 | API、SDK、架构文档、开发者生态 |
| 代码仓库搜索 | 开源项目、README、提交频率、issue、star、依赖 |
| 用户云文档搜索 | 用户已有飞书资料、历史投研文档、访谈纪要 |

## 信息类型路由

### 公司基础信息

| 信息 | 工具能力 | 查询词模板 |
|------|----------|------------|
| 公司概况 | 通用网页搜索 / 创投信息搜索 | `{公司名} 公司概况 业务模式 产品` |
| 融资历史 | 创投信息搜索 / 通用网页搜索 | `{公司名} 融资 估值 投资方 轮次` |
| 团队/创始人 | 通用网页搜索 | `{公司名} 创始人 CEO 团队 背景` |
| 财务数据 | 通用网页搜索 / 产业信息搜索 | `{公司名} 营收 利润 财务数据 增长` |

### 产品与技术信息

| 信息 | 工具能力 | 查询词模板 |
|------|----------|------------|
| 产品功能 | 通用网页搜索 | `{公司名} 产品 功能 客户 评测` |
| 技术栈/架构 | 技术文档搜索 / 通用网页搜索 | `{公司名} 技术架构 技术栈 API SDK` |
| 开源项目 | 代码仓库搜索 / 通用网页搜索 | `{公司名} GitHub open source README` |
| API/开发者生态 | 技术文档搜索 | `{公司名} API docs SDK developer` |
| 技术性能基准 | 通用网页搜索 / 技术文档搜索 | `{公司名} benchmark 性能测试 评测 对比` |

### 行业与竞争信息

| 信息 | 工具能力 | 查询词模板 |
|------|----------|------------|
| 行业报告 | 通用网页搜索 / 产业信息搜索 | `{行业名} 行业报告 市场规模 {当前年份} {前一年}` |
| 竞品排名 | 通用网页搜索 / 创投信息搜索 | `{行业名} 主要玩家 排名 市占率` |
| 用户评价 | 通用网页搜索 | `{公司名} 评价 体验 优缺点 客户` |
| 用户已有资料 | 用户云文档搜索 | `{公司名}`、`{行业名}`、`竞对分析` |

## 语言策略

- `zh-first`：中文查询优先，适合中国 PE/VC 项目、人民币基金、国内产业链研究。
- `en-first`：英文查询优先，适合海外公司、全球市场、英文技术生态。
- `auto`：根据目标公司所在地、客户市场、技术生态自动判断。

同一事实尽量用中英文资料交叉验证；如果只能找到单语种资料，在报告中标注信息局限。

## 频率与并行限制

竞对分析可以并行搜索多个竞对，但每个子代理必须只负责一个竞对或一个主题。

每个子代理默认限制：

- 3-5 个查询
- 只读证据收集
- 不写最终报告
- 不写飞书文档
- 不发送消息
- 不向用户请求确认
- 搜索失败时返回部分结果和信息缺口

返回格式遵循 `search-policy.md` 中的“并行搜索约束”。
