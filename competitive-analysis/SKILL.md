# 竞对分析

对投资目标公司与竞争对手进行系统性对比分析，评估竞争威胁，为投资决策提供竞争维度的判断。

**核心原则**：PE 投资需要理解竞争格局——不知道对手是谁，就无法验证竞争壁垒的真实性。技术驱动型公司的竞对分析必须深入技术维度——技术架构、技术栈、开源依赖、研发投入决定了竞争的底层逻辑。

---

## 工作流

### 第零步：读取配置与协作模式

先读取统一配置，再决定搜索工具和输出方式。配置策略参考：

- 搜索工具配置：`references/search-policy.md`
- 协作模式配置：`references/collaboration-policy.md`
- 飞书操作指南：`references/lark-collaboration.md`

配置读取顺序：

1. 优先读取 `config/investment-research-config.json`
2. 其次使用本 skill 运行期配置；如果缺失则首次运行时询问用户
3. 如果搜索工具或协作模式仍为空或为 `ask`，首次运行时询问用户选择，并写入 `config/investment-research-config.json`

协作模式：

| 模式 | 输入 | 输出 |
|------|------|------|
| 本地模式 | 目标公司 + 竞对列表 | 本地 Markdown 文件 |
| 飞书协作模式 | 目标公司 + 竞对列表 + 可选飞书资料 | 飞书云文档 + 本地备份 |

如果配置为飞书协作模式：
1. 询问是否有飞书云文档需要参考（行业报告、竞品分析材料等）
2. 使用 `lark-cli docs +fetch` 读取飞书文档内容
3. 与搜索结果合并

如果飞书工具不可用或权限不足，回退到本地 Markdown 输出，不中断研究。

### 第一步：确认分析范围

1. **确认目标公司**：名称、行业、核心业务
2. **确认竞对列表**：
   - 用户提供的竞对名单
   - 如果用户未提供，基于行业研究推荐 3-5 个主要竞对
3. **确认分析深度**：
   - 快速扫描（2-3个竞对，每个30分钟）
   - 深度分析（3-5个竞对，每个1小时+）

### 第二步：行业特征发现

在构建对比框架前，先分析行业以识别关键竞争因素。

1. **行业分类**：确定目标公司的行业层级，尽可能具体
2. **识别关键成功因素（KSF）**：
   - 买方决策标准：客户如何选择供应商？
   - 竞争壁垒类型：什么创造持久优势？
   - 行业特有指标：哪些运营指标最重要？
   - 监管/政策因素：哪些法规塑造竞争？
   - 技术颠覆风险：哪些新兴技术可能重塑行业？
3. **维度权重分配**：基于 KSF 发现，为六维画像分配权重

对比框架参考：`references/comparison-framework.md`

### 第三步：信息搜索

对每个竞对进行多源信息搜索。

#### 3.1 基础信息搜索

按 `references/search-policy.md` 选择用户配置的通用搜索工具。建议查询主题：

| 信息 | 搜索策略 |
|------|---------|
| 公司基本信息 | "XX公司 融资 估值 团队" |
| 产品信息 | "XX公司 产品 功能 客户" |
| 财务信息 | "XX公司 营收 增长 利润" |
| 融资信息 | "XX公司 融资轮次 投资方" |

#### 3.2 技术深度搜索

按 `references/search-policy.md` 选择用户配置的技术搜索工具。建议查询主题：

| 信息 | 搜索策略 |
|------|---------|
| 技术架构 | 查技术文档、架构设计文档 |
| 开源项目 | 仓库结构、star 数、最近提交频率 |
| 技术性能基准 | "XX benchmark 性能测试 评测" |
| 研发团队 | "XX公司 研发团队 CTO 技术负责人" |

**技术搜索要点**：
- 如果竞对有开源项目，优先使用当前配置中支持代码仓库读取的工具查看仓库结构、README、最近提交
- 如果竞对使用知名框架/库，优先使用当前配置中支持技术文档读取的工具查询文档
- 技术性能基准搜索需包含"评测"、"对比"、"benchmark"等关键词
- 研发团队搜索需包含 CTO、技术负责人、核心工程师

### 第四步：六维画像构建

对每个公司（目标+竞对）从六个维度构建画像，并基于行业特征调整权重。

#### 4.1 基础六维画像

| 维度 | 默认权重 | 关键指标 |
|------|---------|---------|
| 产品 | 20% | 核心产品、产品形态、目标客户、功能覆盖 |
| 技术 | 15% | 技术栈、核心技术壁垒、开源项目、专利 |
| 商业模式 | 20% | 收费模式、定价、客户规模、渠道 |
| 团队 | 15% | 创始人背景、团队规模、核心人才 |
| 融资 | 15% | 最新轮次、估值、投资方、融资节奏 |
| 财务 | 15% | 营收规模、增长率、盈利状态、现金流 |

权重根据行业特征发现结果调整。

#### 4.2 技术维度深度子维度

技术维度从简单的"技术栈+壁垒"扩展为6个深度子维度：

| 子维度 | 评估内容 | 评分依据 |
|--------|---------|---------|
| 技术架构 | 单体/微服务/事件驱动/混合 | 架构先进性和扩展性 |
| 技术栈 | 语言/框架/基础设施选择 | 技术栈成熟度和生态 |
| 开源vs专有 | 开源依赖比例、自研核心比例 | 自研比例越高壁垒越强 |
| 研发投入 | 研发团队规模、研发支出占比 | 投入越大迭代越快 |
| 技术成熟度 | 技术方案的工程化程度 | 从POC到大规模生产的成熟度 |
| 技术路线图 | 未来技术演进方向 | 路线图清晰度和可行性 |

**技术子维度评分标准**：

| 星级 | 技术架构 | 技术栈 | 开源vs专有 | 研发投入 | 技术成熟度 | 技术路线图 |
|------|---------|--------|-----------|---------|-----------|-----------|
| ★ | 单体架构，扩展困难 | 过时技术栈 | 100%开源依赖 | 研发<5%收入 | POC阶段 | 无路线图 |
| ★★ | 初步模块化 | 部分过时 | 80%开源 | 5-10%收入 | 小规模验证 | 模糊方向 |
| ★★★ | 微服务/标准架构 | 主流技术栈 | 50%自研 | 10-15%收入 | 规模化部署 | 有路线图但模糊 |
| ★★★★ | 先进架构，良好扩展 | 前沿+成熟混合 | 70%自研 | 15-20%收入 | 大规模生产 | 清晰路线图 |
| ★★★★★ | 创新架构，行业领先 | 引领技术趋势 | 90%+自研 | 20%+收入 | 行业标杆 | 清晰+前瞻 |

#### 4.3 行业特定子维度

根据行业特征发现结果，为每个维度添加行业特有的子维度和评分标准。

常见行业模式参考：`references/comparison-framework.md` 中的"常见行业模式"部分。

### 第五步：对比分析与威胁评估

#### 5.1 维度对比

将所有公司的六维画像进行横向对比，识别：
- 目标公司的优势维度
- 目标公司的劣势维度
- 竞对领先的关键维度

#### 5.2 技术维度深度对比

特别关注技术维度的深度对比：
- 技术架构差异：谁的架构更先进？扩展性更好？
- 技术栈差异：谁的技术选型更合理？生态更完善？
- 自研比例差异：谁的自研核心更多？壁垒更强？
- 研发投入差异：谁的研发更激进？迭代更快？
- 技术成熟度差异：谁的工程化程度更高？离大规模生产更近？
- 技术路线图差异：谁的方向更清晰？更有前瞻性？

#### 5.3 威胁评估

| 威胁等级 | 条件 | 含义 | 投资建议 |
|---------|------|------|---------|
| 🔴 高威胁 | 综合评分 ≥ 目标 且 产品高度重叠 | 竞对可直接替代 | 需差异化突围，否则投资风险高 |
| 🟡 中威胁 | 部分维度领先 但 整体不及 | 竞对在特定领域有优势 | 关注动态，利用对方弱项 |
| 🟢 低威胁 | 整体弱于目标公司 | 竞对暂不构成直接威胁 | 保持优势，防止追赶 |

### 第六步：生成报告

基于以上分析，生成结构化的竞对分析报告。

**报告结构**：

1. **执行摘要**：竞争格局概览、核心发现、威胁评估
2. **行业特征**：关键成功因素、维度权重
3. **公司画像**：每个公司的六维画像（含技术深度子维度）
4. **对比分析**：维度横向对比、技术深度对比
5. **威胁评估**：每个竞对的威胁等级和理由
6. **投资建议**：基于竞争分析的投资建议

**输出方式**：
- 本地模式：写入 Markdown 文件
- 飞书协作模式：
  - 追加到已有飞书文档：`lark-cli docs +update --api-version v2 --doc <token> --command append --content '<XML>'`
  - 创建新飞书文档：`lark-cli docs +create --api-version v2 --content '<XML>'`
  - 发送文档链接：`lark-cli im +messages-send --user-id <open_id> --as user --markdown "报告链接: <url>"`
  - 同时本地保存 Markdown 备份

## 输出契约

最终输出包含以下部分：

1. **竞争格局概览**：行业竞争态势和关键发现
2. **六维画像对比表**：所有公司的维度评分对比
3. **技术深度对比**：技术6子维度的详细对比
4. **威胁评估矩阵**：每个竞对的威胁等级
5. **投资建议**：基于竞争分析的投资建议

## 注意事项

- 竞对信息的可靠性参差不齐，每条信息需标注来源和可靠性等级
- 技术维度搜索可能受限于公开信息，信息缺失标注 "—"，不参与评分
- 超过1年的数据标注"可能过时"
- 技术对比需要同口径比较，避免不同条件下的性能数据直接对比
- 威胁评估是动态的，当前评估基于当前信息，需定期更新
