# 行业研究

对目标投资行业进行系统性研究，构建行业全景图，识别投资机会和风险，提炼投资逻辑。

**核心原则**：PE 行业研究的目标是回答"这个赛道值不值得投"和"投什么类型的企业"。技术驱动型行业的研究必须包含核心技术拆解——技术是行业演进的核心驱动力，也是投资机会的核心来源。

## 工作流

### 第零步：读取配置与协作模式

先读取统一配置，再决定搜索工具和输出方式。配置策略参考：

- 搜索工具配置：`references/search-policy.md`
- 协作模式配置：`references/collaboration-policy.md`
- 飞书操作指南：`references/lark-collaboration.md`

配置读取顺序：

1. 优先读取 `config/investment-research-config.json`
2. 其次使用本 skill 运行期配置；如果缺失则首次运行时询问用户
3. 如果搜索工具或协作模式仍为空或为 `ask`，首次运行时询问用户选择，并写入 `config/investment-research-config.json`

协作模式：

| 模式 | 输入 | 输出 |
|------|------|------|
| 本地模式 | 行业关键词 | 本地 Markdown 文件 |
| 飞书协作模式 | 行业关键词 + 可选飞书资料 | 飞书云文档 + 本地备份 |

如果配置为飞书协作模式：
1. 询问是否有飞书云文档需要参考（行业报告、研究材料等）
2. 使用 `lark-cli docs +fetch` 读取飞书文档内容
3. 与搜索结果合并

如果飞书工具不可用或权限不足，回退到本地 Markdown 输出，不中断研究。

### 第一步：行业定义与边界

明确研究的行业范围和边界。

1. **行业分类**：确定行业层级，尽可能具体
   - 大行业 → 子行业 → 细分领域
   - 例：医疗 → 数字健康 → B2B SaaS 远程医疗平台
   - 例：制造 → 新能源汽车 → 动力电池

2. **行业定义**：用一句话定义这个行业
   - 这个行业解决什么问题？
   - 为谁解决？
   - 怎么解决？

3. **行业边界**：明确包含和排除的领域
   - 相邻行业有哪些？边界在哪里？
   - 本研究的范围是否包含上下游？

**搜索策略**：按 `references/search-policy.md` 选择用户配置的通用搜索工具，搜索"XX行业 定义 分类 范围"、行业研究报告、白皮书等资料。

### 第二步：市场规模与增长

量化行业的市场空间和增长前景。

1. **市场规模**：
   - TAM（总可触达市场）/ SAM（可服务市场）/ SOM（可获取市场）
   - 当前市场规模（金额、增速）
   - 数据来源和计算逻辑

2. **增长驱动**：
   - 核心增长驱动因素（需求端、供给端、政策端）
   - 增长的可持续性分析
   - 增长的天花板在哪里？

3. **增长预测**：
   - 短期（1-2年）/ 中期（3-5年）/ 长期（5-10年）增长预期
   - 预测的假设和不确定性

**搜索策略**：按 `references/search-policy.md` 选择用户配置的通用搜索工具，搜索"XX行业 市场规模 TAM 增速"、第三方行业报告、协会/政府统计、咨询机构数据等资料。

### 第三步：竞争格局地图

绘制行业竞争格局，识别关键玩家和竞争态势。

1. **市场参与者分类**：
   - 按业务模式分类
   - 按规模分类（龙头/腰部/初创）
   - 按资本属性分类（上市公司/非上市/外资）

2. **竞争格局分析**：
   - 市场集中度（CR3/CR5/HHI）
   - 竞争强度和竞争维度
   - 行业处于什么竞争阶段（分散→集中→寡头）

3. **关键玩家画像**：
   - Top 5-10 玩家的基本信息
   - 各自的定位和差异化
   - 估值/融资情况

**搜索策略**：按 `references/search-policy.md` 选择用户配置的通用搜索工具，搜索"XX行业 竞争格局 龙头 排名"、行业排名、市场份额数据等资料。

### 第四步：行业核心技术拆解

**这是技术驱动型行业研究的核心步骤。**

对行业依赖的核心技术进行系统性拆解，理解技术如何驱动行业演进。

1. **技术栈分解**：行业依赖的核心技术组件和层次
   - 例：AI → 基础模型层 / 推理引擎层 / 应用层
   - 例：半导体 → 设计EDA / 制造光刻 / 封测
   - 例：SaaS → 基础设施层 / 平台层 / 应用层
   - 每层的关键技术、成熟度、演进方向

2. **架构模式**：行业主流技术架构和设计范式
   - 行业通用的架构模式
   - 新兴架构趋势
   - 架构选择对业务的影响

3. **关键技术挑战**：行业面临的核心技术难题
   - 当前技术瓶颈
   - 正在攻关的方向
   - 突破可能带来的行业变革

4. **技术成熟度映射**：各子领域在技术成熟度曲线上的位置
   - 哪些技术已成熟并大规模应用？
   - 哪些技术处于快速成长期？
   - 哪些技术仍处于早期探索？
   - 技术成熟度对投资时机的影响

5. **开源 vs 专有生态**：行业技术生态的开源/商业化比例
   - 核心开源项目及其影响力
   - 专有技术的壁垒来源
   - 开源对商业化的影响（赋能 vs 替代）

**搜索策略**：按 `references/search-policy.md` 选择用户配置的技术搜索工具，搜索"XX行业 核心技术 技术栈 技术架构"、核心开源项目、技术文档、技术评测、技术趋势报告等资料。

### 第五步：价值链分析

分析行业价值链，识别价值创造和利润分配的关键环节。

1. **价值链结构**：
   - 上游：原材料/基础技术/基础设施
   - 中游：核心产品/服务/平台
   - 下游：渠道/客户/应用场景

2. **价值分配**：
   - 各环节的附加值和利润率
   - 价值链中的利润池在哪里？
   - 价值分配的变化趋势

3. **技术控制点**：
   - 价值链中哪些环节由技术能力决定？
   - 技术壁垒如何影响价值分配？
   - 技术创新如何改变价值链结构？

价值链画布参考：`references/value-chain.md`

### 第六步：政策与监管

分析影响行业的政策法规和监管环境。

1. **监管框架**：
   - 行业主要监管机构和法规
   - 准入门槛和牌照要求
   - 合规成本

2. **政策趋势**：
   - 近期重要政策变化
   - 政策方向和趋势
   - 政策对投资的影响

3. **技术监管**：
   - 技术相关的监管要求（数据安全、算法合规等）
   - 技术标准认证要求
   - 监管对技术创新的影响

### 第七步：技术趋势

分析行业技术演进方向和未来趋势。

1. **当前技术格局**：
   - 主流技术路线和代表企业
   - 技术路线之争（如有）

2. **新兴技术趋势**：
   - 可能改变行业格局的新技术
   - 技术演进的时间线
   - 技术替代的风险和机会

3. **技术投资机会**：
   - 技术变革催生的新投资机会
   - 技术落后者被淘汰的风险
   - 技术拐点的判断

> 注意：第四步"核心技术拆解"侧重行业当前的技术结构，第七步"技术趋势"侧重未来演进方向。

### 第八步：投资逻辑提炼

综合以上分析，提炼行业投资逻辑。

1. **投资机会**：
   - 最有吸引力的投资方向
   - 推荐的企业类型和特征
   - 投资时机的判断

2. **投资风险**：
   - 行业系统性风险
   - 技术路线风险
   - 政策风险
   - 竞争风险

3. **技术投资逻辑**：
   - 技术壁垒是投资的核心护城河
   - 技术成熟度决定投资时机
   - 技术路线选择决定投资成败
   - 研发能力是长期竞争力的保障

4. **关键判断**：
   - 这个赛道值不值得投？
   - 投什么类型的企业？
   - 什么时机投？
   - 需要规避什么风险？

行业画布参考：`references/industry-canvas.md`

## 输出契约

最终输出包含以下部分：

1. **行业定义与边界**：清晰的行业范围界定
2. **市场规模与增长**：量化的市场数据和增长逻辑
3. **竞争格局地图**：关键玩家和竞争态势
4. **核心技术拆解**：技术栈、架构、挑战、成熟度、生态
5. **价值链分析**：价值创造和分配，技术控制点
6. **政策与监管**：监管框架和政策趋势
7. **技术趋势**：技术演进方向和投资机会
8. **投资逻辑**：投资机会、风险、关键判断

**输出方式**：
- 本地模式：写入 Markdown 文件
- 飞书协作模式：
  - 追加到已有飞书文档：`lark-cli docs +update --api-version v2 --doc <token> --command append --content '<XML>'`
  - 创建新飞书文档：`lark-cli docs +create --api-version v2 --content '<XML>'`
  - 发送文档链接：`lark-cli im +messages-send --user-id <open_id> --as user --markdown "报告链接: <url>"`
  - 同时本地保存 Markdown 备份

## 注意事项

- 行业研究的核心是"建立投资框架"，不是"罗列所有信息"
- 核心技术拆解是科技行业研究的核心步骤，不能省略
- 市场规模数据需标注来源和计算逻辑，不同来源的数据可能差异很大
- 竞争格局需要动态视角，关注趋势而非静态快照
- 投资逻辑必须基于分析，区分"事实"和"判断"
- 技术趋势分析需要区分"已发生的趋势"和"预测的趋势"
